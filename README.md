# mudora-analysis

Analysis of item placement in [A Link to the Past Randomizer](https://alttpr.com) (ALttPR), based on a dataset of ~28k randomized games generated by the [PHP randomizer](https://github.com/sporchia/alttp_vt_randomizer) and inspected with [Mudora](https://github.com/alttpr-mudora/mudora).

![Item locations for progression](/assets/item_location_heatmap_progression_only_scaled.png)

## What's here

- [dataset.ipynb](dataset.ipynb) - build a dataset from your own randomized games.
- [main.ipynb](main.ipynb) - performs analysis over the dataset (loaded from Kaggle).
- Mudora binary not included, grab one here: [https://github.com/alttpr-mudora](https://github.com/alttpr-mudora)

## Setup

You'll need your own legally-acquired unheadered Japanese v1.0 ALttP ROM to generate randomized seeds — none are included here.

1. Set up the [PHP randomizer](https://github.com/sporchia/alttp_vt_randomizer) and generate a batch of ROMs, e.g.:

   ```sh
   php artisan alttp:randomize <ROM> <output_dir> \
     --goal=ganon --state=open --weapons=randomized --glitches=none \
     --crystals_ganon=7 --crystals_tower=7 --item_placement=advanced \
     --dungeon_items=standard --accessibility=items --hints=on \
     --item_pool=normal --item_functionality=normal
   ```

2. Acquire a `mudora.exe` binary and verify it works: `./mudora.exe -v`.
3. Copy the env template and point it at your generated ROMs:

   ```sh
   cp .env.example .env
   # set ROM_PATH=/path/to/roms
   ```

4. Run `dataset.ipynb` to produce `alttpr.csv`, then `main.ipynb` to reproduce the analysis (or just load the published dataset from Kaggle, as `main.ipynb` does by default).
